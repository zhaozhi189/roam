# Roam 需求说明书 v0.1

> 作者:zhaozhi · 起草日期:2026-05-12 · 状态:草稿(MVP 设计)

## 1. 产品定位

### 1.1 一句话

**Roam(即境)** 把"扫一个真实空间 → 浏览器里走进去"变成人人都能用的工具,分享一个 URL 朋友手机点开就能漫游。

### 1.2 背景

- 2026-05 月一条推文爆红:开发者用手机扫房子 → 训练 3D Gaussian Splatting(3DGS)模型 → PlayCanvas 里浏览器漫游看房
- 关键技术拐点:**SplatTransform 2.0** 让 3DGS 从"看得见摸不着的全息影像"变成"有物理碰撞的可交互空间"
- **制作成本砍掉 90%+**:以前几周建模师工作,现在手机 30 分钟搞定
- 类似时机:**3D 互联网的"HTML 时刻"**——复杂 3D 变成发个链接一样简单

### 1.3 不做什么(明确边界)

- ❌ 不做大模型应用 / AI Agent(MVP 阶段)
- ❌ 不做建模工具,不是 Blender / SketchUp 的替代
- ❌ 不做游戏引擎,不是 Unity / Unreal 的替代
- ❌ 不做付费 SaaS 商业产品(至少前 6 个月)
- ✅ 是**空间扫描 + 浏览器漫游 + 一键分享**的轻工具

## 2. 用户与场景

### 2.1 目标用户(MVP 阶段)

| 类型 | 描述 | 优先级 |
|---|---|---|
| 个人玩家 | 自己想玩玩 3D 扫描,做点有意思的链接分享给朋友 | P0 |
| 摄影 / 设计爱好者 | 把作品空间数字化,做作品集 | P1 |
| 房产中介 / 小工作室 | 看房链接发给客户 | P2(不主推) |

### 2.2 核心用户故事

**Story 1 · 周末扫家**
- 我是普通用户,用 Android 手机扫了我家客厅
- 上传到 Roam,等几分钟训练完
- 拿到一个 URL,微信发给朋友
- 朋友手机点开,在浏览器里像玩游戏一样走进我家
- 走到沙发会被挡住,走到墙边会停下来

**Story 2 · 留念**
- 我搬家了,扫一下老房子留个数字记忆
- 多年以后还能"走"回来

**Story 3 · 朋友圈炫技**
- 我去了一个有意思的咖啡馆,扫一下
- 转发到朋友圈:"你们也来走一走"

## 3. 功能需求

### 3.1 MVP(P0,目标 1-2 周内可用)

| ID | 功能 | 说明 |
|---|---|---|
| F-01 | **场景上传** | 用户上传已经训练好的 `.ply` / `.sog` 文件(MVP 不做训练) |
| F-02 | **场景列表** | 用户能看到自己上传的所有场景,缩略图 + 名字 |
| F-03 | **浏览器漫游** | 第一人称 WASD + 触屏摇杆,跑得动 PC 和 Android Chrome |
| F-04 | **物理碰撞** | 加载场景时附带的 collision.glb,墙壁地板能撞 |
| F-05 | **分享 URL** | 每个场景一个公开 URL,免登录访问 |
| F-06 | **移动端适配** | Android Chrome / iOS Safari 都能开,自动降低画质保流畅 |

### 3.2 Phase 2(P1,MVP 验证 OK 后)

| ID | 功能 | 说明 |
|---|---|---|
| F-07 | **端到端扫描** | 用户在 App 内一键扫描 → 自动调云训练 → 自动产出可漫游链接 |
| F-08 | **多场景拼接** | 把多个 splat 拼成一个大场景(一栋房子多个房间) |
| F-09 | **截图 / 录屏** | 漫游时一键拍照 / 录视频,分享到社交 |
| F-10 | **打点 / 标注** | 在场景里某个位置插小标签("这里是我读书的角落") |
| F-11 | **PWA 安装** | Android Chrome 可"添加到主屏幕",像 App 一样启动 |
| F-12 | **基础统计** | 我的场景被多少人看过,看了多久 |

### 3.3 Phase 3(P2,不在当前规划)

- F-13 多人同场景实时漫游(WebSocket)
- F-14 LLM 加持(AI 导游、自然语言搜场景、自动标注房间类型)
- F-15 文生 3D 改装预览
- F-16 Google Play / App Store 上架

## 4. 非功能需求

### 4.1 性能

| 指标 | 目标 |
|---|---|
| 单场景文件大小 | ≤ 30 MB(Streamed SOG 流式加载) |
| 冷启动到可漫游 | 移动端 ≤ 5 秒,PC ≤ 2 秒 |
| 漫游帧率 | 旗舰机 60fps,千元机 ≥ 24fps |
| 服务端响应 | URL 打开后首屏 < 1 秒 |

### 4.2 兼容性

| 平台 | MVP 要求 |
|---|---|
| Android Chrome | ✅ 主目标,Pixel 6 / 一加 8 及以上必须流畅 |
| iOS Safari | ✅ iPhone 11 及以上必须能跑(性能可降级) |
| PC Chrome / Edge | ✅ |
| PC Firefox | ⚠️ 可用即可,不投入优化 |

### 4.3 隐私 / 法律

- 用户上传内容只服务于该用户和被他显式分享的人
- **不做**人脸识别 / 人物身份分析
- **不做**精确定位 / 地理坐标关联
- 上传时给用户清晰提示:"请确保扫描的空间和物品归你所有或已获授权"
- 提供"一键删除"入口,数据彻底删干净

### 4.4 安全

- 上传文件类型严格白名单(.ply / .sog / .glb)
- 单用户上传频率限制(防滥用)
- URL 默认公开,但提供"非公开链接"选项(带 token)
- 不存储 PII(个人身份信息)

## 5. 技术选型

### 5.1 整体架构(MVP)

```
[Android 手机]
   ↓ 用户用 KIRI Engine / Polycam / Luma AI 自己扫,导出 .ply
[用户电脑]
   ↓ (可选)本地跑 SplatTransform 2.0 处理出 .sog + collision.glb
[Roam Web 应用]
   ├─ 前端: PlayCanvas Engine + 自研 UI / 第一人称控制器
   ├─ 后端: Node.js / Bun 上传 + 元数据 API
   └─ 存储: Cloudflare R2 / 阿里云 OSS(splat + glb 文件)
[浏览器]
   ↓ 一个 URL 打开
   渲染 + 漫游 + 分享
```

**说明**:MVP 阶段不做云训练,让用户**自己扫好上传**,降低后端复杂度和成本。

### 5.2 技术栈

| 层 | 选型 | 理由 |
|---|---|---|
| 前端框架 | Vite + TypeScript | 个人项目最快上线 |
| 3D 引擎 | **PlayCanvas Engine**(MIT 开源) | 原文方案,移动端优化好 |
| Splat 处理 | **SplatTransform 2.0** | PlayCanvas 配套,生成碰撞体 |
| UI 框架 | React 或 Solid.js(待定) | 看个人偏好 |
| 路由 | 任意轻量(wouter / react-router) | |
| 后端 | Node.js / Bun + Hono | 上传 + 元数据 API |
| 数据库 | SQLite(MVP 起步) | 单机够用 |
| 文件存储 | Cloudflare R2 | 免费额度大,CDN 内置 |
| 部署 | Cloudflare Pages + Workers | 全免费起步 |
| 监控 | Sentry 免费版 | |

### 5.3 不选什么(及理由)

| 没选 | 理由 |
|---|---|
| Three.js | 比 PlayCanvas 更底层,移动端优化要自己做 |
| Unity / Unreal | 个人玩玩太重 |
| LLM(任何) | MVP 不需要;以后做 AI 导游再加 |
| 自建训练流水线 | MVP 让用户用 Polycam / Luma 自己出 .ply |
| 微信小程序 | WebGL 性能受限,长期看不如 Web App |
| React Native | 引入复杂度高,纯 Web 已足够 |

### 5.4 文件流转约定

```
用户扫描产物    →  scene.ply       (KIRI / Polycam / Luma 导出)
            ↓
本地处理(可选)  →  splat-transform scene.ply -K scene.sog
            ↓
            产出 scene.sog (压缩 splat) + scene.collision.glb (碰撞网格)
            ↓
Roam 上传      →  R2:  /scenes/{uuid}/scene.sog
                       /scenes/{uuid}/scene.collision.glb
                       /scenes/{uuid}/thumb.jpg(自动生成)
            ↓
Roam 元数据    →  SQLite:
                  scenes(id, owner_id, name, created_at, size_bytes, public_url)
```

## 6. 里程碑

| 阶段 | 时间 | 交付 |

## 8. 风险与缓解

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| Android 千元机跑不动 PlayCanvas | 中 | 高 | LOD + 帧率自适应 + 最低机型门槛声明 |
| Cloudflare R2 免费额度不够 | 低 | 中 | 文件大小严格控制 + 用户量小时不会触发 |
| 用户扫描出的 splat 质量差,体验差 | 高 | 中 | 写一篇"如何拍出好 splat"指南 |
| PlayCanvas 引擎 API 学习曲线 | 中 | 中 | 第一周专门花时间跑官方 Demo |
| 个人精力不足 | 高 | 高 | 控制 MVP 范围,F-07 之后的功能不在 6 周内 |
| 没人用 | 中 | 低 | 玩玩项目本来就不指望;朋友圈分享拉几个种子用户 |
