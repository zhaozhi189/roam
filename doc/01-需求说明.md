## 1. 产品定位

### 1.1 一句话

**Roam(即境)** 让你扫描一个真实空间 → **手机里漫游 → 录视频 → 微信发给朋友**。MVP 阶段是一款 **Android 单机 App**,不依赖互联网。

### 1.2 背景

- 2026-05 月一条推文爆红:开发者用手机扫房子 → 训练 3D Gaussian Splatting(3DGS)模型 → PlayCanvas 里浏览器漫游看房
- 关键技术拐点:**SPZ 格式开源** + Scaniverse 设备端处理 + PlayCanvas runtime 直接加载 SPZ,**全流程不需要离开手机**
- **制作成本砍掉 90%+**:手机 30 分钟扫完 + 漫游分享
- 类似时机:**3D vlog 时刻**——把 3D 空间变成普通短视频一样易传播

### 1.3 不做什么(明确边界,v0.2 大改)

- ❌ **不做 Web SaaS 公开链接平台**(ADR-004,本地优先)
- ❌ **不做云训练 / 后端 API**(ADR-004,无服务器)
- ❌ **不做用户登录系统**(本地 App 无身份)
- ❌ **不做 iOS**(ADR-005,Android-only)
- ❌ 不做大模型应用 / AI Agent
- ❌ 不做建模工具,不是 Blender / SketchUp 替代
- ❌ 不做付费 SaaS(至少前 6 个月)
- ✅ 是**手机扫描 + 手机漫游 + 录视频 + 微信分享**的轻工具

## 2. 用户与场景

### 2.1 目标用户(MVP)

| 类型 | 描述 | 优先级 |
|---|---|---|
| 个人玩家 | 扫家 / 工作室 / 出去玩的有趣空间,**录视频发朋友圈**炫技 | P0 |
| 摄影 / 设计爱好者 | 把作品空间数字化做"3D vlog" | P1 |
| ~~房产中介~~ | v0.2 移除 — 公开 URL 没了,B 端价值丢失 | — |

### 2.2 核心用户故事(v0.2 大改)

**Story 1 · 周末扫家发视频**
- 我用 Magic7 Pro Scaniverse 扫了客厅(60-90s 设备端处理)
- 导出 .spz 文件,Roam App 选这个文件导入
- 在 App 里漫游,用第一人称走一圈,**点录制按钮录 60s 视频**
- 点分享,系统弹微信选择,发给朋友
- 朋友在微信看到一段流畅的 3D 漫游视频,惊叹"你家长这样啊"

**Story 2 · 留念归档**
- 我搬家了,扫一下老房子留个数字记忆
- 存在手机本地场景库,过几年还能"走"回来录段感慨视频

**Story 3 · 朋友圈炫技**
- 去了有意思的咖啡馆,扫一下
- 录段流畅运镜视频发朋友圈,**接收方就当短视频看**(看不出是 3D)
- 行家会问:"这是 3DGS?",**这就是种子用户的入口**

> ⚠️ **接收方不漫游**(ADR-004 取舍),只看视频。M2 阶段 Q4 决定是否做"扫码回路"让朋友也能装 Roam 自己漫游。

## 3. 功能需求(v0.2 重写)

### 3.1 MVP(P0,目标 4-6 周)

| ID | 功能 | 说明 |
|---|---|---|
| F-01 | **本地场景导入** | Storage Access Framework 选 .spz 文件 → 复制到 internal storage |
| F-02 | **本地场景库** | Room DB 元数据 + 列表 UI:封面 + 名字 + 创建时间 + 大小 |
| F-03 | **场景重命名 / 删除** | 长按场景 → 操作菜单 |
| F-04 | **第一人称漫游** | WebView + PlayCanvas + `@spz-loader/playcanvas` 加载 SPZ;触屏摇杆 + 拖拽视角;**MVP 不做物理碰撞**(可穿墙,见 Q8) |
| F-05 | **录制漫游视频** | MediaProjection 录全屏(含 UI),输出 1080p@30fps H.264 MP4,**默认 60s 上限**(适配微信) |
| F-06 | **分享视频** | Share Intent + FileProvider → 系统分享面板 → 用户选「微信」→ 发给朋友 / 朋友圈 |
| F-07 | **录制中相机控制** | 录制中仍可用第一人称控制(走动 / 转向),不冻结 |
| F-08 | **APK 自分发** | zhi 自签名 APK,M4 阶段给朋友装 |

### 3.2 Phase 2(P1,MVP 验证 OK 后)

| ID | 功能 | 说明 |
|---|---|---|
| F-09 | **录制运镜路径** | 预设关键点路径(类似 PowerPoint 切换),**自动运镜**录视频(主理人不用手动走) |
| F-10 | **打点 / 标注** | 在场景里某个位置插小标签 |
| F-11 | **截图 / GIF** | 单帧截图 + 短 GIF 导出(微信表情包) |
| F-12 | **物理碰撞** | 重新启用,看 Q8 决定方案 |
| F-13 | **PWA / Web 公网版**(回归 Cloudflare 退路) | 让接收方点链接也能漫游(打通 Q4 接收方回路) |
| F-14 | **多场景拼接** | 把多个 splat 拼成一栋房子 |
| F-15 | **iOS 版** | 用 Capacitor / Tauri Mobile 做跨端 |

### 3.3 Phase 3(P2,长期)

- F-16 多人同场景实时漫游
- F-17 LLM 加持(AI 导游、自然语言搜场景)
- F-18 文生 3D 改装预览
- F-19 应用市场上架(华为 / 小米 / Magic 应用市场)

## 4. 非功能需求

### 4.1 性能(v0.2 更新)

| 指标 | 目标 | 已验证情况 |
|---|---|---|
| 单场景文件大小 | SPZ ≤ 30MB(典型 10-12MB) | Scaniverse 实测 |
| App 冷启动到能漫游 | ≤ 2 秒(本地无网络) | M1 第 0 周 H9 验证 |
| SPZ 加载耗时 | ≤ 3 秒 | M1 第 0 周 H9b 验证 |
| 漫游帧率(Magic7 Pro) | ≥ 60fps | H1/H2 已验证 Chrome,WebView 待 H9 |
| 录制中漫游帧率 | ≥ 30fps | M1 H7 验证 |
| 60s 1080p 视频文件大小 | ≤ 25MB(微信限制) | M1 H7 验证 |

### 4.2 兼容性(v0.2 大幅收窄)

| 平台 | MVP 要求 |
|---|---|
| Android Chrome WebView | ✅ 主目标,Magic7 Pro 必须流畅 |
| Android 中端机(2-3 年前) | ⏳ 不在 MVP,Phase 2 评估 |
| ~~iOS Safari~~ | ❌ ADR-005 明确不做 |
| ~~PC 浏览器~~ | ❌ MVP 是 Android App |

### 4.3 隐私 / 法律(v0.2 大幅简化)

- ✅ **数据全在本地**,不上传任何云,不需要隐私政策
- ✅ 无身份系统 / 无 PII 收集
- ✅ 无网络遥测
- ⚠️ MediaProjection 每次录屏弹系统权限提示(Android 强制)
- ⚠️ 用户扫描他人空间需自行确保有权

### 4.4 安全(v0.2 大幅简化)

- ✅ 文件白名单:仅接受 `.spz`(SPZ 格式)
- ✅ APK 自签名分发,Android 系统提示"未知来源",zhi 主动允许
- ⚠️ 手机丢失风险:本地存储不加密(MVP 不做,M2 评估)

## 5. 技术选型(v0.2 大改,详见 03-技术架构.md)

### 5.1 整体架构

```
[Layer 1 · 手机扫描]                  [Layer 2 · Roam Android App]
  Scaniverse 扫房间               ─→   WebView + PlayCanvas + JS Bridge
  导出 .spz(11MB)                      ├ 本地场景库(Room DB)
                                       ├ @spz-loader/playcanvas 加载
                                       ├ 漫游
                                       ├ MediaProjection 录屏
                                       └ Share Intent → 微信
```

### 5.2 技术栈

| 层 | 选型 | 理由 |
|---|---|---|
| App 形态 | **Android Activity + WebView**(Kotlin) | ADR-005 |
| 前端栈 | HTML / CSS / Tailwind CDN(MVP)→ Vite + 框架(M3+) | 复用 prototypes |
| 3D 引擎 | **PlayCanvas Engine** | H1/H2 已验证 |
| SPZ 加载 | **`@spz-loader/playcanvas`** | runtime 直接加载,跳过 SplatTransform |
| 第一人称控制 | PlayCanvas 内置 + 触屏摇杆 | |
| JS Bridge | `WebView.addJavascriptInterface` | 暴露原生能力 |
| 本地存储 | Android internal storage + Room DB | |
| 文件导入 | Storage Access Framework | |
| 视频录制 | **MediaProjection API** | 录全屏,含 UI |
| 微信分享 | **Android Share Intent** + FileProvider | 不要微信 OpenSDK(无企业资质) |
| 分发 | 自签名 APK + adb install | M4 给朋友装 |

### 5.3 不选什么(v0.2 更新)

| 没选 | 理由 |
|---|---|
| Cloudflare R2 / Workers / D1 / Pages | ADR-004 转向本地;保留 wrangler 配置作 Phase 2 退路 |
| GitHub OAuth / 任何登录系统 | 本地 App 无身份 |
| 微信 OpenSDK | 个人开发者无企业资质;Share Intent 够用 |
| iOS / Capacitor / RN / Tauri Mobile 跨端 | iOS 不在 MVP |
| Three.js | PlayCanvas 已验证流畅 |
| Unity / Unreal | 个人项目过重 |
| LLM | 不需要,见 [04 § 1.3](04-上下文交接.md) |
| 自建训练 / 云训练 | ADR-004 推迟 |
| **SplatTransform 主流程** | Q7 关闭,SPZ 直通;**仅 H9 失败时才回退** |

### 5.4 文件流转(v0.2 简化)

```
[手机 Scaniverse]
  扫描 → scene.spz(~10-12MB)
       ↓ 留在手机文件系统

[手机 Roam App]
  Storage Access Framework 选 .spz
       ↓ 复制到 internal storage
  Room DB 写元数据(scene_id, name, file_path, size, created_at)
       ↓
  WebView + PlayCanvas + @spz-loader 加载渲染
       ↓
  MediaProjection 录屏 → MP4
       ↓
  Share Intent 分享给微信
```

## 8. 风险与缓解(v0.2 更新)

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| `@spz-loader/playcanvas` 在 Android WebView 跑不通 | 中 | 高 | M1 H9b 早验证;失败回退 SplatTransform 主路径 |
| Android System WebView ≠ Chrome,性能落差大 | 低 | 高 | M1 H9a 早验证;失败用 X5 内核 |
| MediaProjection 录制中 PlayCanvas 帧率掉太多 | 中 | 中 | 降到 720p / 24fps;限制录制时长 |
| 视频文件大小超过微信 25MB 限制 | 中 | 中 | 主动降码率;切片;最差用 7zip 压缩(尴尬) |
| 物理碰撞缺失导致漫游体验差 | 中 | 中 | Q8 评估;最坏 Phase 2 局部回归 SplatTransform |
| Android Studio + Kotlin 学习曲线 | 高 | 中 | 让 Claude 写带详细中文注释的代码;zhi review + 实跑测试 |
| Bus factor = 1 + 1(Claude + zhi) | 高 | 中 | 关键模块写最小可运行 demo;高频文档同步 |
| 没人用 | 中 | 低 | 玩玩项目本来不指望;朋友圈分享种子用户 |
