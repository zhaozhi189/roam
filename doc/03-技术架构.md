## 二、3 层结构说明(v0.2 从 4 层简化)

### Layer 1 · 手机扫描端

- **设备**:Magic7 Pro(zhi 主力机)+ 同类近 2 年 Android 旗舰
- **App**:**Scaniverse**(Niantic,免费,设备端处理,绕过 WorkOS) — 已装到 Magic7 Pro 验证可用
- **产物**:`scene.ply`(原始 3D Gaussian Splatting)
- **备选**:KIRI Engine Pro($17.99/月,如果 Scaniverse 实际质量不满意)

### Layer 2 · 本地压缩(⚠️ 体验断点,待 M1 重新评估)

- **机器**:**zhi 自己的电脑**(MacBook)
- **工具**:SplatTransform 2.0(`@playcanvas/splat-transform`,Node CLI)
- **关键命令**:`splat-transform scene.ply -K scene.sog`(一次产物 = 压缩 splat + 碰撞 GLB)
- **产物**:`scene.sog`(< 30 MB)、`scene.collision.glb`

> **⚠️ 体验冲突**:Layer 1 在手机,Layer 3 在手机,但 Layer 2 卡在电脑。主理人扫完要把 .ply 从手机传电脑跑命令,再传回手机。**这是 ADR-004 转向本地后产生的新问题,记为 Q7,M1 第 0 周重新评估**。可能方案:
> - A. 接受体验断点,M1 阶段先就这么用(工程量 0)
> - B. 把 SplatTransform 移植到 Android(大工程,~1-2 周额外)
> - C. 让 Roam Android App 直接吃原始 .ply(性能未验证,可能内存爆)
> - D. 找 Scaniverse 能不能直接导出 .compressed.ply(可能跳过 Layer 2)

### Layer 3 · Roam Android App(MVP 核心)

WebView 套壳形态,详见 [ADR-005](decisions/ADR-005-应用形态-Android-WebView套壳.md)。

| 模块 | 实现 |
|---|---|
| **应用主体** | Android Activity + WebView(Kotlin) |
| **前端** | HTML / CSS / JS 放 `assets/`,WebView 加载本地路径;Tailwind CDN(M1 可换 Vite + 框架) |
| **3D 引擎** | PlayCanvas Engine,跑在 WebView 内(H2 验证 Magic7 Pro Chrome 流畅,H9 待验证 WebView 同等性能) |
| **第一人称控制** | PlayCanvas 内置控制器 + 自定义触屏 / 虚拟摇杆 |
| **JS Bridge** | `WebView.addJavascriptInterface()` + Kotlin `@JavascriptInterface`,暴露原生能力给前端 |
| **本地存储** | Android internal storage(`/data/data/com.roam.app/files/`)存 .sog;Room DB 或 SharedPreferences 存场景元数据 |
| **文件导入** | Storage Access Framework(`ACTION_OPEN_DOCUMENT`)选 .sog → 复制到 internal storage |
| **视频录制** | MediaProjection API 录全屏(含 WebView 渲染 + UI)→ MP4 |
| **分享** | `Intent.ACTION_SEND` + FileProvider 视频 URI → 系统分享面板 → 用户选「微信」 |
| **分发** | **自签名 APK**,zhi `adb install` 装到 Magic7 Pro。**MVP 不上应用商店** |

### ~~Layer 4 浏览器消费端~~ → 取消

ADR-004 转向本地后,接收方在微信看视频,不再有独立的浏览器消费端。**Phase 2 如果做公网分享,这一层可能回归**(见下方 Phase 2 退路)。

### Phase 2 退路 · Cloudflare 全家桶(暂不启用)

ADR-004 决策时**故意保留**这一层的可用性:

| 组件 | 状态 |
|---|---|
| Cloudflare 账号 | ✅ 已注册 |
| Wrangler CLI 4.90.1 | ✅ 已装并 login |
| R2 / Pages / Workers / D1 | ❌ MVP 不启用 |

**何时启用**:
- M3-M4 阶段如果发现**视频微信分享的传播链太短**(用户希望"接收方点链接自己漫游"),启用 R2 + Pages 做公网漫游
- 这时新增 Layer 4 浏览器消费端 + Layer 5 Cloudflare 服务端

## 三、数据流转(MVP 路径,本地闭环)

```
[手机扫描]
  Scaniverse → scene.ply

[传到电脑 ⚠️ 体验断点 Q7]
  scene.ply → SplatTransform → scene.sog + collision.glb

[传回手机 ⚠️ 同上]
  scene.sog → Storage Access Framework 选择导入

[手机 Roam App 内]
  ├ 场景库列表(本地 Room DB)
  ├ 点开场景 → WebView 加载 PlayCanvas → 3D 漫游
  ├ 录制按钮 → MediaProjection 录 60s
  └ 分享按钮 → Share Intent → 微信 → 发给朋友

[接收方]
  微信对话 → 看视频 → (可选 Q4 二维码回路)
```

## 五、关键技术选型理由(v0.2)

| 选型 | 理由 |
|---|---|
| **Android WebView 套壳** | ADR-005 比较 4 个形态选定。复用 prototypes 前端,工程量 ~600 行 Kotlin |
| **PlayCanvas Engine** | 移动端优化好,MIT 开源;H1/H2 已验证流畅;WebGL 渲染兼容 WebView |
| **SplatTransform 2.0** | PlayCanvas 配套,一行命令产出 splat + 碰撞,无替代 |
| **Scaniverse** | 免费 + 设备端 + .ply 导出 + 国内可访问(对比 Polycam/Luma WorkOS 拒) |
| **MediaProjection 录全屏** | 包含 PlayCanvas 渲染 + UI 操作,接收方看的视频更完整;对比 canvas.captureStream 只录渲染画面 |
| **Share Intent 分享** | 不需要微信 OpenSDK 企业认证;对比 PWA 的 Web Share API 几乎一样体验 |
| **Kotlin + Android Studio** | 原生 Android 最稳;对比 Tauri Mobile / RN / Capacitor 等跨端方案,Android-only MVP 不需要跨端开销 |
| **不上 Cloudflare** | ADR-004 转向本地;保留 Wrangler 配置作 Phase 2 退路 |
| **不上 LLM** | 不变,见 [04 § 1.3](04-上下文交接.md) |

## 六、性能关键点(v0.2)

| 关注点 | 当前方案 |
|---|---|
| 单场景文件大小 | ≤ 30 MB(SplatTransform 压缩) |
| 启动速度 | APK 本地,无网络冷启动;WebView 加载本地 HTML,~1 秒可漫游 |
| WebView 渲染 | H9 待验证 — WebView 是否真等同 Chrome;预期 Magic7 Pro WebView ≥ 60fps |
| 视频录制不掉帧 | H7 待验证 — 1080p@30fps 录 60s,录制中 PlayCanvas 仍 ≥ 30fps |
| 视频文件大小 | 60s 1080p H.264 ≤ 25MB(微信限制);H7 验证码率控制 |
| 内存 | 30MB .sog 加载到 GPU 后大约 100-200MB 显存;Magic7 Pro 12GB+ RAM 完全不愁 |

## 七、安全 / 隐私(v0.2 大幅简化)

ADR-004 转向本地后,**几乎所有原 v0.1 隐私顾虑消失**:

- ✅ **无服务器存储**:扫描数据全在用户手机,不上传任何云
- ✅ **无身份系统**:没有登录,不需要 OAuth,不收集用户信息
- ✅ **无网络遥测**:MVP 不收 analytics(M2 阶段如果要看用户行为再考虑本地埋点 + 自愿上报)
- ✅ **微信分享是用户主动**:Share Intent 由用户点按钮触发,不是后台偷偷上传
- ⚠️ **MediaProjection 权限提示**:Android 每次录屏弹「即将开始捕获屏幕上的所有内容」对话框,系统级保护,用户必须主动确认

**唯一新隐私点**:用户扫的房间数据存本地,如果手机丢了 / 被借走,有泄露风险。**MVP 不加密**(过度工程),M2 再评估是否加密 internal storage。
