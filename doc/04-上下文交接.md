## 一、关键决策权衡(那些没写进 01/02/03 但讨论过的)

### 1.1 为什么 MVP 不做云训练

**讨论上下文**:技术栈里"用户在本地用 SplatTransform 跑 splat-transform 命令"是 MVP 阶段的设计。

**为什么这么选**:
- **后端复杂度大幅降低**:不用接 Polycam / Luma AI API,不用搭训练队列,不用 GPU 调度
- **个人玩玩项目工时极少**:周末晚上工时,做云训练就 3-4 周就过去了
- **用户群体接受度**:目标用户(个人玩家、设计师)能装 Node 跑命令行
- **演进路径清晰**:Phase 2 的 F-07 就是做云训练,**先把核心漫游体验跑通再说**

**为什么不像快手 / 抖音那样傻瓜化扫一键出?**:那是 1000 人月的产品,不是周末项目。

### 1.2 为什么是 Cloudflare 全家桶,不是 Vercel / AWS

**对比过的方案**:
| 选项 | 为什么没选 |
|---|---|
| Vercel + S3 | Vercel 函数冷启动慢,S3 出口流量贵 |
| AWS Amplify | 学习曲线陡,免费额度过期就贵 |
| 阿里云 / 腾讯云 | 国内付费门槛 + 备案麻烦,个人玩玩不值 |
| 自建 VPS | 运维负担 |
| Cloudflare 全家桶 ✅ | **R2 免费 10GB + 无出口费**,Pages 免费,Workers 免费,D1 SQLite 兼容,**起步零成本,跨 region 自带 CDN** |

**关键判断**:个人项目最大杀手不是技术,是**月底信用卡账单**——Cloudflare R2 没有出口流量费这一条就赢了。

### 1.3 为什么坚决不上 LLM

**讨论过的应用场景**(都被拒了):
- AI 导游 / 房产顾问问答 — Phase 3 才考虑
- 自然语言搜场景 — MVP 用户量小,直接搜文件名足够
- 自动场景标注 — 让用户自己起名字
- 多语言介绍生成 — MVP 中文够了
- 文生 3D 改装预览 — 2026 技术尚不成熟

**底层逻辑**:**LLM 是产品体验放大器,不是基础设施**。3DGS + PlayCanvas 是"基础设施",这条路本来就站得住。把 LLM 放进 MVP = 拖延上线 = 项目死亡风险拉满。

### 1.4 为什么没选其他 7 个名字

完整决策已在 [02-命名过程.md § 四](02-命名过程.md) 固化,核心是:**轻盈、即时、不需要解释 → Roam / 即境**。

### 1.5 为什么 Android 优先,iOS 降级

**讨论过的视角**:
- 我自己用 Android(zhi 的主力机)
- Android Chrome 在国内可控性更高(不像 iOS Safari 那么受 Apple 限制)
- iOS 第二位,**目标可用即可,不为它额外投入**

**注意**:技术架构里 Layer 4 把 iOS Safari 标了"降级流畅 · 内存严格",这是清醒的边界,不是含糊承诺。

### 1.6 为什么编号 NN-标题.md,不是 yyyy-mm-dd-标题.md

**讨论过的选项**:
- 日期前缀:每天创建会很乱,而且文档不是按"产出日"组织的
- 章节号(1.1, 1.2, 1.3):嵌套太深,Roam 项目还没到这个体量
- 简单 NN- 编号:**目前用这个**,简单 + 可见阅读顺序 + 不预设结构

## 二、对你工作风格的观察(给未来 Claude 看)

这些是 2026-05-12 一次会话里观察到的、稳定的偏好:

### 2.1 文档组织

- **中文为主**,术语保留英文(PlayCanvas / 3DGS / R2 这种不要硬翻)
- 章节用 `## 一、` `## 二、` 中文序号(在 aidoc 偶尔混用 `## 1.` 阿拉伯,容忍但偏好中文)
- 表格密集,**能列表表格化的不要散叙**
- 每个大段落要有 "Why" / "How to apply" / "不做什么" 三类小节
- 修订记录写在文末

### 2.2 决策记录习惯

- **决策的反面要写出来**(为什么不选 X、为什么不选 Y),这是 02-命名过程.md 里被明确赞许的格式
- 风险表(影响 × 概率 × 缓解)是标配
- 开放问题列表里要有 "倾向" 列,不要假装中立

### 2.3 协作偏好

- 重要操作会确认("可以"/"好的"/"OK"作为放行信号)
- 喜欢被告知"接下来三个选择",而不是被强制执行单一方案
- 偏好我先沉淀,他再调整;不喜欢被反复反问"你确认吗"
- 对**冗余流程零容忍**:能 inline 解决就不要建子文档

### 2.4 工程价值观

- 个人项目坚决**控制范围**,不追求大而全
- 商业化排在体验后面("至少前 6 个月不商业化")
- 对"AI 加持 / LLM 包装"保持冷静,**不为热点而引入**
- 偏好已验证的方案(PlayCanvas 是因为爆款推文里跑通了)
