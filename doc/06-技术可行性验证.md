## 一、目的与适用范围

### 1.1 为什么先做可行性

Roam 的核心价值链是「**3DGS 文件 → 浏览器 → 流畅漫游**」。其中任何一环不成立,产品就不成立。**M0 的唯一目标是用最低成本证明这条链路能跑通**。

### 1.2 M0 通过 ≠ MVP 完成

M0 通过 = 5 个关键假设全部 ✅(v0.2 移除 H4),可以放心进入工程;**不等于** 写完产品代码。M0 阶段的产物是 **demo + 数据 + 结论**,不是可用产品。

### 1.3 与 03-技术架构.md 的分工

- `03-技术架构.md`:**回答"是什么"**(分 4 层、用什么组件)
- 本文:**回答"行不行"**(关键假设能否被打死,以什么为通过标准)

## 三、详细验证矩阵

### H1 · 桌面 Chrome 流畅度(基线)

| 项 | 内容 |
|---|---|
| **完整假设** | PlayCanvas FPS 博客的示例 .sog 场景(<https://blog.playcanvas.com/turning-a-gaussian-splat-into-a-videogame>),在桌面 Chrome 上稳定 ≥ 60fps |
| **测试设备** | MacBook(zhi 主力机)+ 桌面 Chrome 最新版 |
| **测试素材** | **PlayCanvas 官方 FPS 博客自带的 .sog 场景**(不需要自扫) |
| **验证手段** | 1. clone 官方 demo repo<br/>2. `pnpm install && pnpm dev`<br/>3. 打开 `chrome://gpu` 确认硬件加速<br/>4. 用 Chrome devtools Performance 面板录 30 秒 |
| **通过标准** | 平均 ≥ 60fps,p99 < 16.7ms,**无明显掉帧** |
| **预估工时** | 1 小时 |
| **失败兜底** | 如果连官方 demo 都跑不动,**直接放弃 PlayCanvas 选型**,改回评估 Three.js + gsplat.js;此为最坏情况,概率 < 5% |

### H2 · Magic7 Pro 满血基线

| 项 | 内容 |
|---|---|
| **完整假设** | 同一份官方 .sog,在 zhi 的 Magic7 Pro(骁龙 8 Gen4 + Adreno 830)上稳定 ≥ 60fps |
| **测试设备** | zhi 的 Magic7 Pro(主力机,旗舰) |
| **验证手段** | 1. H1 dev server 跑起来,通过局域网 IP 让手机 Chrome 访问<br/>2. 加 `stats.js` 显示实时 fps<br/>3. 用 Chrome remote debugging 看 Performance |
| **通过标准** | ≥ 60fps,无明显掉帧 |
| **预估工时** | 1 小时 |
| **失败兜底** | 旗舰过不去 = 重新评估 PlayCanvas 移动端能力(概率极低) |

### ~~H2b · 中端机底线(推迟)~~

不在 M0 验证。等 H2 通过 + M1 阶段再决定方案:借中端机 / Chrome devtools 4× CPU + 6× GPU 节流模拟 / 降级 MVP 承诺为「近 2 年旗舰」。

### H3 · iOS Safari(可选,not blocker)

| 项 | 内容 |
|---|---|
| **完整假设** | iPhone 11(2019 款,作为底线) Safari 能加载官方示例 .sog 并漫游 |
| **测试设备** | zhi 没有 iPhone → 借设备或用 BrowserStack 远程 |
| **验证手段** | 1. H1 dev server 加 ngrok 暴露公网 URL<br/>2. iOS Safari 访问<br/>3. 观察是否 OOM 崩溃,fps 是否 ≥ 24 |
| **通过标准** | 官方场景不崩,fps ≥ 24 |
| **预估工时** | 1 小时(主要消耗在借设备) |
| **失败兜底** | iOS 在落地页明确标"实验性,部分机型可能不可用";不做承诺 |

### ~~H4 · SplatTransform 压缩比(推迟到 M1)~~

**v0.2 移除**。M0 不验证自扫场景。理由见 § 八。

M1 阶段补做时:
- 工具优先级:**Scaniverse(Niantic)免费,设备端处理,Android 支持,.ply 导出,绕过 WorkOS 拦截** > KIRI Pro $17.99/月(备选)> 其他工具(Luma 国内 policy_denied / Polycam Android 体验差 / postshot 与 Reality Scan 仅 Windows)
- 下载途径:Play Store 国内不可用,用 **APKPure / Uptodown** 镜像下 APK
- 验证场景:zhi 卧室,~10㎡
- 通过标准:产物 ≤ 30MB,视觉劣化在主观可接受

### H5 · R2 + 流式加载首屏耗时

| 项 | 内容 |
|---|---|
| **完整假设** | 把 H1 用的官方示例 .sog 上传 R2,通过 Cloudflare Pages 引用,4G 网络模拟下从「点链接」到「可漫游」**< 10 秒** |
| **测试素材** | **PlayCanvas 官方示例 .sog**(不依赖自扫) |
| **测试环境** | Chrome devtools Network 节流到 Fast 4G(4Mbps DL,3 Mbps UL) |
| **验证手段** | 1. 把 .sog 用 `wrangler r2 object put` 上传 R2<br/>2. Pages 部署最小 demo 加载该 R2 URL<br/>3. devtools 测 `DOMContentLoaded` + `splat ready event` |
| **通过标准** | 官方示例场景大小(实测决定),4G 节流下 < 10s 可漫游;p95 < 15s |
| **预估工时** | 1 小时 |
| **失败兜底** | 慢于 15s:加 PlayCanvas 渐进 LOD(先加载低密度版,后台流式补齐);最坏:在 P1 落地页加显式 loading 进度条 |

### H6 · 上传链路打通(端到端)

| 项 | 内容 |
|---|---|
| **完整假设** | 写一个最小后端:GitHub OAuth 登录 → 上传 .sog 文件(用官方示例)→ 写 R2 → 写 D1 → 返回公开 URL,**全链路 < 200 行代码** |
| **验证手段** | 1. `wrangler init` 创建 Workers 项目<br/>2. Hono 加 2 个路由:`POST /api/upload` 和 `GET /api/auth/github/callback`<br/>3. D1 建一张 `scenes` 表(uuid, owner, name, size)<br/>4. 用 curl 或 Postman 走通 |
| **通过标准** | 端到端 demo 跑通,文件落 R2,元数据落 D1,公开 URL 可访问 |
| **预估工时** | 2 小时(主要在 Cloudflare 控制台配置) |
| **失败兜底** | OAuth 配置卡住:先用 mock token 走完链路;D1 卡住:用 R2 元数据 JSON 替代;**Hono 卡住:换原生 Workers handler** |

## 七、总验收标准(v0.2)

M0 通过 = 以下全部满足:

- [ ] H1 桌面 ≥ 60fps 通过
- [ ] H2 Magic7 Pro ≥ 60fps 通过
- [ ] H5 官方示例 .sog 上传 R2,4G 模拟 < 10s 可漫游
- [ ] H6 上传链路端到端跑通,可手动用 curl 上传一个文件并通过返回 URL 浏览
- [ ] H3 iOS 通过或显式标记"实验性"(可选,不阻塞总验收)

**M0 失败的判定**:任何一项 H1/H2 失败 = 整体失败,回到方案评估;H3/H5/H6 失败 = 调整边界后继续推进。

## 八、不验证什么 / 推迟到 M1(v0.2)

| 项 | 理由 |
|---|---|
| ⏳ **H4 自扫 + SplatTransform 压缩**(M1) | Luma / Polycam 国内访问受 WorkOS 策略限制,KIRI 3DGS 需 Pro 订阅($17.99/月)。这是「钱 + 网络环境」问题,不是「技术不确定性」 — 留到 M1 验证管线完成、决定真要扫场景时再处理 |
| ⏳ **H2b 真实中端机底线**(M1) | zhi 备用机即旗舰,无法直接测中端;借机 / BrowserStack / Chrome 节流模拟方案延后决定 |
| ❌ 云端训练(.ply 直接进系统) | F-07 是 Phase 2,MVP 用户在本地自己跑 SplatTransform |
| ❌ PWA 安装体验 | F-11 Phase 2,MVP 是普通网页 |
| ❌ 多场景拼接 / 大场景流式 | Phase 2/3,MVP 单场景 ≤ 30MB |
| ❌ LLM 任何用法 | 永久不在 MVP,见 [04-上下文交接.md § 1.3](04-上下文交接.md) |
| ❌ 商业化 / 付费链路 | "至少前 6 个月不商业化" |
| ❌ 多人协作 / 实时同步 | Phase 3 |
| ❌ 千元机以下机型 | 边界明确,不投入精力 |
