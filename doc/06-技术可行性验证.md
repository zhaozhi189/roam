## 三、详细验证矩阵

### ✅ H1 · 桌面 Chrome 流畅度(已通过)

| 项 | 内容 |
|---|---|
| **完整假设** | PlayCanvas FPS 博客 .sog 场景在桌面 Chrome ≥ 60fps |
| **测试设备** | MacBook + Chrome 147.0.7727.102,Mac OS X 26.4.1 |
| **测试素材** | <https://playcanv.as/p/qxGSuzYq/>(PlayCanvas 公网托管 demo) |
| **实际结果** | GPU 全硬件加速(WebGL / WebGL2 / WebGPU / Compositing / Rasterization 全绿);主观漫游流畅,移动无问题 |
| **结论** | ✅ 通过(2026-05-12) |

### ✅ H2 · Magic7 Pro 满血基线(已通过)

| 项 | 内容 |
|---|---|
| **完整假设** | Magic7 Pro(骁龙 8 Gen4 + Adreno 830)跑同一 demo ≥ 60fps |
| **测试方式** | 手机 Chrome 直接打开公网 demo URL |
| **实际结果** | **比 MacBook 还流畅**,zhi 主观评价 |
| **结论** | ✅ 通过(2026-05-12) — Adreno 830 在 3DGS 场景上确实强 |

### ✅ H7 · 视频录制(WebCodecs + mp4-muxer,已通过)

| 项 | 内容 |
|---|---|
| **完整假设(最终方案)** | WebView 内用 `canvas.captureStream` + `MediaStreamTrackProcessor` + `VideoEncoder(avc1.42E01F)` + mp4-muxer 直出 H.264 MP4,**绕开 MediaProjection 死循环** |
| **为什么不用 MediaProjection** | MagicOS 不尊重 `createConfigForDefaultDisplay()`,授权页默认「单个应用」让 Roam 切到 launcher → 录到空文件;每次都要重新授权(系统强制)。详见 [ADR-006](decisions/ADR-006-录屏方案-WebCodecs-mp4-muxer.md) |
| **实际结果** | ✅ 录制 5-10s 文件 1.5-3MB / 1920×1080 等比 / 30fps / 3Mbps H.264 Baseline;0 授权弹窗;录制秒响应;微信可正常播放 |
| **依赖** | `vendor/mp4-muxer.min.js` v5.2.2(31KB IIFE,挂 `window.Mp4Muxer`);WebCodecs API(Android Chrome/WebView 102+) |
| **取舍** | 只录 canvas 内容,不录 UI 手指 — 接收方在微信看到「纯 3D 漫游 Vlog」反而更好 |
| **失败兜底**(Phase 2 退路保留) | `ScreenRecorderService.kt` + FOREGROUND_SERVICE_MEDIA_PROJECTION 权限**全保留**,M2 如发现老设备 WebCodecs 不可用 → 降级 MediaProjection |
| **结论** | ✅ 通过(2026-05-15) |

### ✅ H8 · 微信 Share Intent(已通过)

| 项 | 内容 |
|---|---|
| **完整假设** | Android `Intent.ACTION_SEND` 带视频文件 URI,微信会出现在系统分享面板,选「微信」→「发送给朋友」→ 选联系人 → 视频成功发送到对话框,**对方能正常播放** |
| **实际结果** | ✅ FileProvider + `${applicationId}.fileprovider` authority 配置成功;微信出现在分享面板;视频成功发到对话框;**WebM 格式微信不能播放**(关键发现 → 驱动 ADR-006 切到 WebCodecs MP4);改 MP4 后 ✅ 正常播放 |
| **依赖** | `androidx.core.content.FileProvider` + `res/xml/file_paths.xml` 暴露 `recordings/` 子目录 |
| **结论** | ✅ 通过(2026-05-15)— 同时澄清了「微信视频兼容」隐含约束 = H.264 MP4 |

### ✅ H9a · WebView 内 PlayCanvas 性能(已通过)

| 项 | 内容 |
|---|---|
| **完整假设** | Magic7 Pro Android System WebView 里跑 PlayCanvas 立方体场景,fps ≥ 60 |
| **实际结果** | ✅ **119 fps 实测**(立方体场景),远超 60fps 标准;PlayCanvas Engine 2.18.1 UMD vendor 到 `assets/vendor/playcanvas.min.js`(2.2MB,挂 `window.pc`)— 因 jsdelivr CDN 国内不稳改本地 |
| **结论** | ✅ 通过(2026-05-15);Adreno 830 在 WebView 内同样满血 |

### 🟡 H9b · @spz-loader 真渲染(简化版通过,真渲染留 M2)

| 项 | 内容 |
|---|---|
| **完整假设** | `@spz-loader/playcanvas` 0.3.1 在 WebView 加载 .spz 文件 → 渲染 |
| **简化版实测(已做)** | SAF `ACTION_OPEN_DOCUMENT` 选文件 → JS `fetch(content://uri)` → arrayBuffer → 验证前 4 字节 magic `'NGSP'`(SPZ 文件头) → 显示文件大小/字节预览。**整条「Android SAF → WebView → 二进制读取」链路通**,JS 拿到 SPZ raw bytes 没问题 |
| **未做(留 M2)** | `@spz-loader/playcanvas` ESM 包真接入 PlayCanvas 渲染 — ESM/UMD 共存问题 + CDN 国内不稳,留 M2 投入更多时间设计 vendor 方案 |
| **失败兜底** | SPZ loader 真接入失败 → 回退 v0.2 路径:Scaniverse → PLY → SplatTransform → SOG → PlayCanvas |
| **结论** | 🟡 链路通过,渲染留 M2 |
