### 1.2 战略性方向调整(M0 中途发生)

M0 跑到 H5 R2 验证前,zhi 提出转向「本地单机 + 微信视频分享」,触发 4 个关键决策:

| 决策 | 内容 | 影响 |
|---|---|---|
| **ADR-004** | MVP 方向:Web SaaS → 本地 Android App + 视频微信分享 | H5/H6 直接砍 |
| **ADR-005** | 应用形态:Android WebView 套壳 + Share Intent + MediaProjection | 不要微信 OpenSDK,Kotlin 项目 |
| **Q7 关闭** | Scaniverse 直出 SPZ + `@spz-loader/playcanvas` runtime 加载 | Layer 2 SplatTransform 砍掉,主理人全手机闭环 |
| **Q8 关闭** | MVP 物理碰撞砍掉(可穿墙);M2 视违和程度上 PlayCanvas raycaster | F-04 推到 Phase 2 |

## 三、M1 第 0 周开胃菜任务清单

按依赖顺序排列。每项标注**真机依赖**(✅ 不依赖 / ⚠️ 部分 / 🔴 必须)。

### Step 1 · Android 项目骨架(✅ 已完成)

- [x] Android Studio 新建 Empty Activity (Compose) 项目 → `client/`
- [x] 包名 com.roam.app / minSdk 24 / Kotlin DSL + Version Catalog
- [x] settings.gradle.kts 改 `PREFER_SETTINGS` 兼容阿里云镜像
- [x] AndroidManifest.xml 加 INTERNET 权限(M2 vendor 后可移除)
- [x] MainActivity Compose AndroidView 包 WebView
- [x] WebView 配置:javaScript / domStorage / allowFileAccess / allowContentAccess
- [x] assets/index.html H9 测试中心(3 个测试入口)
- [x] gradle assembleDebug 通过(APK 10MB,增量 2s)

**产物**:`client/app/build/outputs/apk/debug/app-debug.apk`

### Step 2 · 真机部署 + H9a 验证(🔴 真机)

- [ ] USB 连 Magic7 Pro,`adb devices` 确认
- [ ] `adb install -r app-debug.apk`
- [ ] `adb shell am start -n com.roam.app/.MainActivity`
- [ ] `adb logcat -s chromium:V Console:V` 抓 WebView 启动日志(WebGL 版本 / GPU 厂商 / UA)
- [ ] index.html 上点 ②「启动 PlayCanvas 场景」,看 FPS(目标 ≥ 60)
- [ ] 点 ①「打开在线 demo」,看 H1/H2 的 demo 在 WebView 是否同等流畅
- [ ] 截图归档到 `M1-results/h9a-webview-perf.md`

**通过标准**:
- WebGL 2.0 可用 ✅
- 立方体 fps ≥ 60
- 在线 demo 主观流畅

### Step 3 · JS Bridge + SAF 选文件(⚠️ 部分,代码可写)

- [ ] Kotlin 端:`RoamBridge.kt` 用 `@JavascriptInterface` 暴露 `pickSpzFile()`
- [ ] Activity 端:`registerForActivityResult(OpenDocument)` 处理 SAF 返回
- [ ] WebView.addJavascriptInterface 注册 RoamBridge
- [ ] JS 端:加 `window.onSpzPicked(uri)` 回调 + UI 显示选中的文件名
- [ ] 真机测:点选文件 → SAF 弹出 → 选个任意文件 → URI 回调到 JS → console.log

### Step 4 · H9b @spz-loader 加载 SPZ(⚠️ 部分,代码可写,真机+SPZ 测)

- [ ] index.html 加 `@spz-loader/playcanvas` CDN(或 npm 装后 vendor 进 assets)
- [ ] 接 Step 3 的 onSpzPicked → `createGSplatEntityFromSpzUrlAsync(app, uri)`
- [ ] **需要 SPZ 文件**:用 Scaniverse 扫一个测试场景 → 导出 SPZ → SAF 选
- [ ] 测加载耗时(目标 < 3s) + 渲染 fps(目标 ≥ 50fps)
- [ ] 截图归档到 `M1-results/h9b-spz-loader.md`

**通过标准**:
- SPZ 文件成功加载,visible 渲染
- 加载耗时 < 3s
- 渲染 fps ≥ 50

**失败兜底**:`@spz-loader/playcanvas` 跑不通 → 退到 v0.2 路径(SplatTransform 跑 PLY → SOG → PlayCanvas 加载 SOG)

### Step 5 · H7 MediaProjection 录全屏(🔴 真机)

- [ ] AndroidManifest.xml 加 `android.permission.FOREGROUND_SERVICE` + `android.permission.FOREGROUND_SERVICE_MEDIA_PROJECTION`
- [ ] 写 `ScreenRecorderService`(ForegroundService + MediaProjection + MediaRecorder)
- [ ] JS Bridge 暴露 `startRecording()` / `stopRecording()`
- [ ] 真机测:60s 1080p@30fps,目标文件 ≤ 25MB,录制中 PlayCanvas fps ≥ 30
- [ ] 截图 + MP4 样本归档到 `M1-results/h7-mediaprojection/`

**通过标准**:文件 ≤ 25MB,视觉无明显劣化,录制中 fps ≥ 30

### Step 6 · H8 Share Intent 微信分享(🔴 真机 + 微信)

- [ ] FileProvider 配置 `app/src/main/res/xml/file_paths.xml` 暴露 internal storage 视频目录
- [ ] AndroidManifest.xml 加 `<provider>` 注册 FileProvider
- [ ] JS Bridge 暴露 `shareVideoToWeChat(filePath)`
- [ ] Kotlin 端构造 `Intent.ACTION_SEND` + `EXTRA_STREAM=FileProvider.getUriForFile(...)` + `type=video/mp4`
- [ ] 真机测:点分享 → 系统面板出现 → 选微信 → 发给自己 → 对方能播放
- [ ] 截图归档到 `M1-results/h8-share-intent/`

**通过标准**:微信成功收到视频,可正常播放,画质对比原片可接受

## 五、给"未来的 zhi / Claude"的提示

### 5.1 拿起这个项目时先看什么

按顺序看 3 个文件,15 分钟入场:

1. **本文档**(07-M0总结):了解 M0 跑到哪 + M1 要做啥
2. **ADR-004 + ADR-005**:理解为什么 MVP 是本地单机 + Android WebView(避免再次质疑方向)
3. **06-技术可行性验证.md v0.4**:看 H1-H9 哪些 ✅ 哪些 ⏳

### 5.2 不要再走的弯路

- ❌ **不要再讨论 Web SaaS / 公开 URL**:ADR-004 已经决断,Phase 2 才会回归
- ❌ **不要再装 KIRI / Polycam / Luma AI**:Scaniverse 已是定论,免费 + SPZ 直出 + 国内可访问
- ❌ **不要再申请微信 OpenSDK**:个人开发者无企业资质,Share Intent 已够
- ❌ **不要再把 SplatTransform 放主路径**:Q7 关闭后只是 H9b 失败的回退方案
- ❌ **不要再纠结 iOS**:ADR-005 明确 Android-only,Phase 2 才考虑跨端

Android Studio  2025.2 (AI-252)

### 5.4 接下来 1 小时能做什么

如果你接手时刚好 1 小时:

1. USB 连 Magic7 Pro
2. `adb install -r client/app/build/outputs/apk/debug/app-debug.apk`
3. 启动 App 跑 Step 2 H9a(15 分钟)
4. 用 Scaniverse 扫一个简单物体(20 分钟,可以扫桌上一个杯子)
5. 等 Step 3 JS Bridge + Step 4 SPZ 加载代码就绪后,跑 H9b(20 分钟)
