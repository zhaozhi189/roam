按依赖顺序排列。每项标注**真机依赖**(✅ 不依赖 / ⚠️ 部分 / 🔴 必须)。

- [x] MainActivity Compose AndroidView 包 WebView
- [x] WebView 配置:javaScript / domStorage / allowFileAccess / allowContentAccess
- [x] assets/index.html H9 测试中心(3 个测试入口)
- [x] gradle assembleDebug 通过(APK 10MB,增量 2s)

**产物**:`client/app/build/outputs/apk/debug/app-debug.apk`

### Step 2 · 真机部署 + H9a 验证(✅ 已完成)

- [x] USB 连 Magic7 Pro,`adb devices` 确认
- [x] `adb install -r app-debug.apk`
- [x] WebView 内 PlayCanvas 立方体 **119 fps**(目标 ≥ 60),远超预期
- [x] 在线 demo 通过 ①入口可访问,主观流畅与 Chrome 等价

**WebView 日志桥接**:由于荣耀 logcat 加密(HKS/HKE),改用 `WebView.setWebContentsDebuggingEnabled(true)` + `onConsoleMessage` 桥接 JS console.log 到 `Roam/WebView` tag,绕过加密

### Step 3 · JS Bridge + SAF 选文件(✅ 已完成)

- [x] `RoamBridge.kt` 暴露 `pickSpzFile()` + `runOnUiThread`
- [x] Activity 端 `registerForActivityResult(OpenDocument)` callback + `evaluateJavascript("window.onSpzPicked('${uri}')")` 回调 JS
- [x] WebView.addJavascriptInterface 注册 `window.RoamBridge`

### Step 4 · H9b SPZ loader(🟡 简化版通过,真渲染留 M2)

- [x] **简化版已通过**:JS 端 `onSpzPicked(uri)` 改 async function → `fetch(content://uri)` → arrayBuffer → 前 4 字节 magic byte 检查 `'NGSP'` → 显示文件大小
- [x] 整条「Android SAF → WebView → 二进制读取」链路通,Roam 拿到 SPZ raw bytes 没问题
- [ ] **未做(留 M2)**:`@spz-loader/playcanvas` ESM 包真接入 PlayCanvas 渲染。ESM/UMD 共存 + CDN 国内不稳问题,需要单独投入时间设计 vendor 方案

**通过标准达成**:文件读取链路 ✅,真渲染待 M2

### Step 5 · H7 录屏(改走 WebCodecs + mp4-muxer,见 ADR-006)

**Y 路径(原计划 MediaProjection)否决**:
- MagicOS 不尊重 `MediaProjectionConfig.createConfigForDefaultDisplay()`,授权页默认「单/整」让用户切到 launcher → Roam 最小化录到空文件(zhi 实测 5 个 0 字节 mp4)
- 每次开始录屏都要重新授权(Android 14+ 强制,所有录屏 App 行为一致)
- 用户反馈「死循环」

**Z 路径(WebCodecs + mp4-muxer)采纳并通过**:
- [x] `vendor/mp4-muxer.min.js` v5.2.2(31KB IIFE)
- [x] `canvas.captureStream(30)` + `MediaStreamTrackProcessor` + `VideoEncoder(avc1.42E01F, 3Mbps)` + Mp4Muxer
- [x] JS Bridge `saveVideoBase64(filename, base64)` 写 internal storage `recordings/`
- [x] 实测 5-10s 录制 1.5-3MB,微信播放 ✅

**Phase 2 退路保留**:`ScreenRecorderService.kt` + FOREGROUND_SERVICE_MEDIA_PROJECTION 权限全保留,M2 如需 → 直接接入

### Step 6 · H8 Share Intent 微信分享(✅ 已完成)

- [x] `res/xml/file_paths.xml` 暴露 `recordings/` 子目录
- [x] AndroidManifest `<provider>` 注册 `${applicationId}.fileprovider`
- [x] `RoamBridge.shareVideoToWeChat(filePath)` + `ShareHelper.shareVideo`(Intent.ACTION_SEND + FileProvider.getUriForFile)
- [x] 实测:点分享 → 系统面板出现微信 → 选微信 → 发给自己 → ✅ MP4 正常播放
- [x] **关键发现**:WebM 微信不能播放(zhi 第一次反馈) → 反推 ADR-006 切到 WebCodecs MP4

### Step 7 · 视频管理(M1 第 0 周补做,✅ 已完成)

收尾发现 App 内体验缺口,补充:
- [x] `RoamBridge.listRecordings()` 列 internal storage mp4/webm
- [x] `RoamBridge.deleteVideo(filePath)` 带 canonical path 越权检查
- [x] `RoamBridge.openVideoExternal(filePath)` Intent.ACTION_VIEW + FileProvider 跳系统视频播放器(WebView 内 `<video src=file://>` 受限)
- [x] index.html 视频列表 ▶播放 / 分享 / 🗑删除三按钮,使用 createElement 避免 innerHTML hook 阻拦

## 五、给"未来的 zhi / Claude"的提示

### 5.1 拿起这个项目时先看什么

按顺序看 3 个文件,15 分钟入场:

1. **本文档**(07-M0总结):了解 M0 跑到哪 + M1 要做啥
2. **ADR-004 + ADR-005**:理解为什么 MVP 是本地单机 + Android WebView(避免再次质疑方向)
3. **06-技术可行性验证.md v0.4**:看 H1-H9 哪些 ✅ 哪些 ⏳

### 5.2 不要再走的弯路

- ❌ **不要再讨论 Web SaaS / 公开 URL**:ADR-004 已经决断,Phase 2 才会回归
- ❌ **不要再装 KIRI / Polycam / Luma AI**:Scaniverse 已是定论,免费 + SPZ 直出 + 国内可访问
- ❌ **不要再申请微信 OpenSDK**:个人开发者无企业资质,Share Intent 已够
- ❌ **不要再把 SplatTransform 放主路径**:Q7 关闭后只是 H9b 失败的回退方案
- ❌ **不要再纠结 iOS**:ADR-005 明确 Android-only,Phase 2 才考虑跨端

Android Studio  2025.2 (AI-252)

### 5.4 接下来 1 小时能做什么

如果你接手时刚好 1 小时:

1. USB 连 Magic7 Pro
2. `adb install -r client/app/build/outputs/apk/debug/app-debug.apk`
3. 启动 App 跑 Step 2 H9a(15 分钟)
4. 用 Scaniverse 扫一个简单物体(20 分钟,可以扫桌上一个杯子)
5. 等 Step 3 JS Bridge + Step 4 SPZ 加载代码就绪后,跑 H9b(20 分钟)
