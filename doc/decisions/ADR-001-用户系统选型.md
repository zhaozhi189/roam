# ADR-001 · 用户系统选 GitHub OAuth

> 状态:✅ Accepted
> 日期:2026-05-12
> 关联:[01-需求说明.md § 7 Q1](../01-需求说明.md)

## 背景 / 上下文

Roam 需要让用户登录后上传场景文件,并能区分"我的场景"vs"别人的场景"。涉及到:

- 用户注册 / 登录
- 密码存储
- 邮件激活 / 找回密码
- 个人玩玩项目,**完全不想运维这套**

## 决策

**用 GitHub OAuth 单一登录,不做密码注册,不做邮箱注册**。

## 备选方案

| 选项 | 评价 | 没选的原因 |
|---|---|---|
| **完全公开**(无用户) | 上传即公开,所有人都能看到所有场景 | 没法标识"我的场景";滥用风险高 |
| **邮箱 + 密码注册** | 完整自有用户体系 | 要做找回密码 / 邮件发送 / 密码哈希;运维负担重 |
| **GitHub OAuth** ✅ | 一键登录;GitHub 自带身份 | **目标用户是开发者/极客,99% 有 GitHub** |
| **Google / Apple OAuth** | 受众更广 | OAuth 配置复杂,且国内访问 Google 不稳 |
| **微信登录** | 国内用户广 | 微信开放平台资质要求高,个人项目搞不来 |

## 后果

### 好的影响

- 零密码运维
- 用户身份从 GitHub 拿,头像 / username 现成
- 用户群体筛选(GitHub 用户群体重叠目标用户)
- Cloudflare Workers + Hono 实现 OAuth 流程 200 行代码搞定

### 不好的影响 / 承担的代价

- **拒绝了 90% 的非开发者用户**(他们没 GitHub 账号)
- 朋友圈分享后,普通用户看不能登录会流失
- **缓解**:看场景**不需要登录**,只有"上传 / 管理我的场景"才需要登录;让公开 URL 对所有人开放

## 何时回头看

- Roam 用户量到 1000 时,如果"非开发者用户登录受阻"的反馈很多,**考虑加 Google OAuth**
- Roam 商业化时,**考虑微信登录**
