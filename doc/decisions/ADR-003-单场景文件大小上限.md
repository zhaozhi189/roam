# ADR-003 · 单场景文件大小上限 30MB

> 状态:✅ Accepted(起步值)
> 日期:2026-05-12
> 关联:[01-需求说明.md § 4.1 + § 7 Q7](../01-需求说明.md)

## 背景 / 上下文

PlayCanvas 官方 FPS Demo 完整包 68MB(含逻辑、贴图、splat),冷启动几秒。Roam 是更轻量的"看房"场景,**只需要 splat + collision.glb + thumb**。

需要为单个上传的场景设定一个**文件大小上限**,平衡:

- 用户体验(移动端流畅度)
- 存储成本(Cloudflare R2 免费 10GB)
- 上传体验(国内移动网络上传大文件慢)

## 决策

**单场景上传上限 30MB**(`.sog` + `.collision.glb` 合计),起步值。

具体拆分建议:

- `.sog`(压缩 splat):**≤ 25MB**
- `.collision.glb`(碰撞网格):**≤ 5MB**
- `thumb.jpg`(后端自动生成):另算,不占用户上限

## 备选方案

| 上限 | 评价 | 没选的原因 |
|---|---|---|
| 10MB | 太严,90% 真实场景超 | 影响产品力 |
| 30MB ✅ | 中位场景能装下,移动端冷启动 5 秒内 | — |
| 50MB | 大场景能装下 | 国内移动上传慢,千元机内存吃紧 |
| 100MB | PlayCanvas 官方 FPS demo 量级 | R2 免费额度撑不住 1000 用户 |
| 无限制 | 用户友好 | 滥用风险 + 成本失控 |

## 后果

### 好的影响

- 30MB 内可保证移动端冷启动 < 5 秒(Cloudflare CDN 走边缘)
- R2 免费 10GB 容量 = 约 350 个场景空间,**单人项目早期足够**
- 用户被迫优化扫描质量(降采样、清噪点),反过来提升观看体验

### 不好的影响 / 承担的代价

- 大场景(整栋房子)装不下,需要 Phase 2 的 F-08 多场景拼接
- 高质量 splat(高斯数百万)无法上传

### 缓解

- 在上传 UI 显式提示:"如果场景较大,请用 SplatTransform 命令行降采样"
- 提供推荐的 SplatTransform 参数(`--voxel-params 0.05,0.1`)
- 给具体场景示例:"中型房间 ~ 8-15MB,整套小公寓 ~ 20-30MB"

## 何时回头看

- M3 移动端调优后,**实测千元机加载 25MB splat 的体验**
- 如果千元机能跑 50MB,上限上调
- 如果千元机 25MB 都吃力,上限下调到 20MB,顺便引入"质量分级"
