# ADR-004 · MVP 方向调整 — 从 Web SaaS 转向本地单机 + 视频分享

> 状态:✅ Accepted
> 日期:2026-05-13(M0 进行中)
> 关联:[01-需求说明.md](../01-需求说明.md)、[03-技术架构.md](../03-技术架构.md)、[06-技术可行性验证.md](../06-技术可行性验证.md)

---

## 背景 / 上下文

M0 阶段技术可行性验证进行到一半,H1(桌面基线)+ H2(Magic7 Pro 旗舰)已通过,**核心技术假设打死**:PlayCanvas + 3DGS 在 zhi 主力设备上完全跑得动,而且 Magic7 Pro 比 MacBook 还流畅。

此时讨论 H5(R2 上传 + 流式加载)前,zhi 提出方向调整:

> "我想先做单机版,可以不需要对象存储,不要互联网,本地有列表,视频可以导出分享,或者微信聊天分享也可以"

这是一次**产品定位的重大转向**,从「Web SaaS 公开链接分享平台」变成「本地 3DGS 浏览器 + 视频导出工具」。

## 决策

**MVP 阶段全本地,不依赖互联网,通过视频导出 + 微信分享**:

- 本地场景库(本地文件系统持久化)
- 本地漫游(浏览器/桌面客户端加载本地 .sog)
- **核心分享方式:导出漫游视频 → 微信/朋友圈分享**
- **不做**:Cloudflare R2 / Workers / D1 / 公开 URL / GitHub OAuth 登录(全部推迟到 Phase 2,如果还要做的话)

## 备选方案

| 选项 | 评价 | 没选的原因 |
|---|---|---|
| **A · 全单机 + 视频分享** ✅ | 0 服务器成本,周末项目压力骤降;微信流量入口最大;离线无依赖 | (选择本项) |
| **B · 本地主体 + 可选公网链接** | 默认本地,用户主动「上传生成公开链接」;两全 | 多套一层 SaaS 链路,违反"控制范围";周末项目精力不足 |
| **C · 纯创作工具(运镜设计器)** | 重点不在漫游,在让用户设计运镜出片 | 偏离 Roam 原定位(漫游分享)太远;受众更窄 |
| **D · 维持原 Web SaaS 方向** | 保留所有原设计 | zhi 已经倾向本地优先,且对运维 / 账号 / 公网部署没兴趣;微信场景被忽视 |

## 后果

### 好的影响

- **0 服务器成本,完全免费**:R2 / Workers / D1 全砍,Cloudflare Free Plan 也不用
- **0 隐私顾虑**:用户数据(扫描场景)全在本地,不上传任何云
- **微信流量入口**:视频比链接传播性强 10 倍,**特别适配国内分享习惯**
- **MVP 工程量减半**:M0 的 H5 / H6 直接砍掉,无需做 OAuth / 后端 API / 部署
- **离线可用**:网络不好的环境也能用
- **GitHub OAuth App 不需要创建**(原计划要做的事直接取消)
- **Cloudflare 注册 / Wrangler 安装 / R2 启用等今晚的投入并未浪费**,推迟到 Phase 2 公网分享时还能用

### 不好的影响 / 承担的代价

- **失去 Web SaaS 网络效应**:接收方不能点链接立刻漫游,只能看视频
- **Roam 命名跟视频分享有违和**:Roam = 漫游,但接收方看视频不漫游,需要重新叙事或改名
- **3DGS 技术魅力被压缩**:视频化以后接收方以为是普通短视频,不知道这是可漫游 3D 场景
- **微信视频 25MB 限制**:高质量 3DGS 录制压缩后画质打折
- **商业化路径变窄**:从「平台型」退化为「工具型」,toC 个人付费比 SaaS 难得多
- **应用形态待定**:本地有多种实现(纯浏览器 HTML / Electron / Tauri / 原生 App),决策复杂度上升

### 缓解措施

- **双角色叙事**:对内(主理人)= 漫游者沉浸体验;对外(微信观众)= 看精彩 vlog
- **视频附文案模板**:导出视频时自带「这是 3DGS 漫游录制,扫码可自己漫游」字幕,引导高级用户回流
- **未来回归 Web SaaS 的退路保留**:今晚的 Cloudflare 工作不删除,Phase 2 真需要公网分享时直接接入

## 何时回头看

- **M1 跑通本地 MVP** 并实测视频分享转化率(发了多少视频,有多少人来问"怎么自己玩")— 用真实数据决定是否回归 Web SaaS
- **如果微信视频压缩严重影响体验**,回头考虑直接做 H5 / H6(本地 + 公网可选 = 选项 B)
- **如果改名讨论收敛到"Roam 仍合适"**,这次决策不影响品牌;如果改名,这个 ADR 是改名决策的上下文起点
