## 背景

1. **触摸交互**(B 触摸控制 / 双击复位 / Back 退沉浸)
2. **场景分享生态**(C deep link / 二维码 / Roam 内扫码 / GitHub Pages landing)
3. **3DGS 格式完整支持**(D 真测 SPZ + .ply/.sog 内置)
4. **生产闭环**(录屏 / 截屏 / MediaStore 导出系统图库)

## 决策

### 1. 触摸控制全套(B 路径)

`attachOrbitTouch(canvas, cam)`:球坐标 (yaw, pitch, radius) 模型 + 单指拖 / 双指捏 / 双击复位 / 3s 无操作恢复自动旋转。`event.preventDefault()` 防 WebView 默认页面滚动。

### 2. 微信生态 deep link 三件套

微信内自定义 scheme 全部被屏蔽(`ERR_UNKNOWN_URL_SCHEME`),三个独立方案:
- **A · ⚠️ 警示 + 🔗 自测按钮**:UI 明示问题,自测调 `RoamBridge.openDeepLink` 直接 `startActivity ACTION_VIEW`
- **B · Roam 内置扫码(ZXing Android Embedded)**:相机扫 → 直接跳同 App 场景,绕开微信
- **C · GitHub Pages landing page**(`site/index.html`):分享 https URL,接收方点 → 微信检测提示「右上角 ⋯ → 浏览器打开」 → `intent://scene/<name>#Intent;scheme=roam;package=com.roam.app;end`

三方案 ship,zhi 根据场景选最合适的(自测用 A/B,真分享用 C 部署后)。

### 3. SPZ 真渲染(D 路径)

vendor `@spz-loader/playcanvas` 0.3.1 UMD(242KB,内嵌 WASM)解决 ESM/UMD 共存难题:UMD 挂 `window.SpzWasmPlayCanvas`,依赖 `window.pc`(已 vendor 的 PlayCanvas Engine)。两者共用同一个 `pc.Application` 实例,Entity 类型兼容。

vendor Niantic 官方 `hornedlizard.spz`(17MB)真测渲染。

### 5. 截屏 + MediaStore 公共导出

`canvas.toDataURL('image/png')` 需要 PlayCanvas Application 加 `graphicsDeviceOptions: { preserveDrawingBuffer: true }`(WebGL 默认不保留 drawing buffer)。

截屏 + 录屏文件双轨写盘:
- `filesDir/recordings/`(App 内列表 ▶/分享/🗑 用)
- `MediaStore.Pictures/Roam/` + `MediaStore.Movies/Roam/`(系统图库直接可见,Android 10+ scoped storage 无需 WRITE 权限)

### 6. 质量基础设施

- **RoamLogic.kt** 提取纯逻辑(无 Android API 依赖)— filename sanitize / 路径越权 / mime / deep link 解析 / JSON escape / 扫码安全决策
- **RoamLogicTest** 46 个 JVM 测试,`./gradlew testDebugUnitTest` 3 秒跑完,JSON escape 用 `org.json:json:20240303` 反向 parse 验证
- **scripts/e2e-test.sh** 端到端 adb 自动化(8 段,截屏归档)
- **adb runAuto broadcast** 暴露 14 个 cmd 给自动化测试

## 备选方案(没选的)

| 选项 | 没选的原因 |
|---|---|
| ML Kit Barcode Scanning(GMS) | Magic7 Pro 国行无 Google Play Services |
| Universal Links / App Links | 需 release 签名稳定 + 域名验证,M2 不投入 |
| Robolectric / 仪表测试 | RoamLogic 抽出后 JVM 直跑就够,Robolectric 启动慢 |
| 自扫工具替代(Polycam Web / 桌面 Postshot) | 用户决策,等 Scaniverse 登录问题解了再说 |
| MainActivity Compose State 重构 | 改动大,YAGNI,现 imperative WebView ref 够用 |

## 不在 M2 范围(明确推 M3+)

- ❌ 接收方未装 Roam 时的下载/安装引导
- ❌ App Links HTTPS 域名验证(需稳定签名 + .well-known/assetlinks.json + 域名)
- ❌ 场景元数据持久化(收藏 / 历史 / 创建时间)
- ❌ 大型自扫场景的流式加载 / LOD
- ❌ 多人协作 / 远程同步
- ❌ AI 导游 / LLM(MVP-Phase 3 都不需要)

## 后果

### 好的影响

- **MVP 完整可用**:朋友实测播放 ✅,所有核心路径有测试或自动化覆盖
- **回归保护**:46 个 JVM 测试守护 RoamLogic 内的安全/逻辑契约,改 RoamBridge/MainActivity 时先看测试
- **新人 onboarding 友好**:README + CLAUDE.md + ADR + e2e 脚本组合让未来 Claude/collaborator 不用从零理解
- **APK 46MB** 合理(+25MB demo 资源换 6 场景丰富度)

### 取舍

- **APK 体积** 从 21MB 涨到 46MB,因 vendor 5 个 demo splat。M2+ 可以让 demo 资源外挂(SAF 选)缩 APK
- **未测部分**:JS UI 交互 / WebCodecs 编码 / ZXing 扫码 / WebViewAssetLoader 路由 — 设备-only 验证(`scripts/e2e-test.sh` 覆盖回归)
- **C 路径**(GitHub Pages)代码就绪但**等 zhi 主动 push 部署**,未上线

### Phase 3 候选(等 zhi 启动)

1. 自扫工具链替代(找一个国内可用的 SPZ/PLY 输出 App)
2. App Links HTTPS 真做(release 签名稳定后)
3. 场景元数据(localStorage / Room)
4. 接收方安装引导(landing page 加 APK 下载链接)
