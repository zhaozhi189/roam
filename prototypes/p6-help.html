如何制作可漫游的 3D 场景

从手机扫描到公开链接,大约需要 30 分钟。本文档随 MVP 持续更新。

Roam 是什么

Roam 是一个 3DGS(3D Gaussian Splatting)场景分享平台。你扫描一个真实空间(比如卧室、咖啡馆、展览),Roam 把它变成一条公开链接,任何人在手机或电脑上都能直接漫游。

核心约束:单场景 ≤ 30 MB,只支持已压缩的 .sog / .compressed.ply。原始 .ply 通常上百 MB,我们不接受。

第 1 步:扫描房间

推荐 App(任选一个):

  • KIRI Engine — Android / iOS 都好用,免费,质量稳
  • Polycam — iOS 优势,扫描手感更顺
  • Luma AI — 训练自动化做得最好

扫描技巧

  • 光照均匀:避免逆光、强阴影
  • 慢速移动:手机每秒移动 ≤ 30cm
  • 多角度:每个物体至少绕 180°
  • 纹理丰富区域优先,纯白墙 / 玻璃尽量少

第 2 步:训练 3DGS

如果你的扫描 App 直接输出 .ply(高斯泼溅格式),跳到第 3 步。否则用以下工具训练:

  • postshot — Windows 桌面端,所见即所得
  • nerfstudio — 开源命令行,需要 GPU

第 3 步:压缩为 .sog

原始 .ply 通常上百 MB,Roam 不接受。用 PlayCanvas 官方 splat-transform 压缩 5-20 倍:

# 安装(Node ≥ 18)
npm install -g @playcanvas/splat-transform

# 压缩
splat-transform input.ply output.sog

典型效果:120 MB → 8 MB(查看对比 demo)。

第 4 步:上传分享

登录 → 「我的场景」 → 「+ 上传新场景」 → 拖入 .sog → 取名 → 完成。系统自动生成公开链接 roam.app/s/xxx。

常见问题

为什么不接受原始 .ply?

原始文件通常 100-500 MB。我们用 Cloudflare R2 免费额度起步,大文件会很快烧光配额。.sog 已经够清晰。

为什么不做云端训练?

MVP 阶段没有 GPU 预算,也不接 Polycam / Luma API。先把"看 + 分享"的体验做好,Phase 2 再考虑端到端扫描。

我可以把场景设为私有吗?

MVP 不行。所有场景默认公开,通过短链 /s/<6-8 位> 访问(不被搜索引擎索引)。
